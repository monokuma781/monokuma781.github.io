 在机群中实现软件再生

 
在机群中实现软件再生

摘　　要

软件故障所引起的系统运转中断远远高于硬件故障，而软件老化是软件故障的一种，它是指软件运行一段时间之后性能下降甚至崩溃的现象，这种现象广泛存在于各种长期运行的软件之中。解决软件老化的重要技术是软件再生技术，这种技术基本思想是在适当的时间终止一个应用，然后清理其内部状态并重新启动这个应用，其特点是在错误发生之前可预先做出反应。在机群系统中引入软件再生技术，对于提高机群系统的软件容错能力和系统可用性具有重要意义。目前，软件再生技术研究的关键问题在于如何确定再生的最佳时机。

本文以曙光4000机群系统为科研背景，以机群操作系统中的系统管理软件为具体对象，深入研究了软件再生技术及其实现方法。创新之处在于：把目前广泛应用于网管领域的基于策略的管理思想和软件再生技术进行结合，扩展了基于策略的管理框架，实现了具有软件再生技术的"曙光4000基于策略的自主管理系统"。具体工作包括：Web服务器Apache软件老化实验的设计和实验数据的统计分析、曙光4000基于策略的自主管理系统的设计和实现。

本文首先介绍了机群的概念和软件再生的理论，同时介绍了本文的主要工作；其次，进行了Web服务器Apache的软件老化实验，分析了实验数据，为"曙光4000基于策略的自主管理系统"的设计和实现提供了依据；再次，描述了"曙光4000基于策略的自主管理系统"的设计思想，以及该系统体系结构和功能结构的设计与实现；之后，分析了该基于策略的自主管理系统的特点，给出了客观的评价；最后，总结全文，提出存在的问题和今后的改进方向。

关键词：机群，基于策略的管理，软件老化，软件再生

Implement Software Rejuvenation in Cluster

Abstract

It is now well established that system outages are caused more due to software faults than due to hardware faults. Software aging is a kind of software fault which is a phenomenon that the state of the software system degrades or crashes with time. This phenomenon exists in all kinds of applications that run a log time. Software rejuvenation is a proactive approach of software fault tolerance which may be used to counteract aging if it exists. Software rejuvenation is based on the idea of gracefully terminating an application and immediately restarting it at a clean internal state. Implementing software rejuvenation in cluster is very important for cluster to enhance software fault tolerance and availability. Now the research in software rejuvenation is focused on how to determine the optimal times to perform rejuvenation.

This paper is based on research of Dawning 4000 Cluster System and summarizes research in software rejuvenation. It is the creative place of the paper that combining policy based management and software rejuvenation, extending the frame of policy based management and implementing Dawning 4000 Policy Based Autonomic Management System with software rejuvenation. The author's concrete work includes designing and analyzing software aging experiment, designing and implementing Dawning 4000 Policy Based Autonomic Management System.

First, the paper introduces the conceptions and related contents of cluster and software rejuvenation; then, software aging in the apache web server will be analyzed and researched; after that, the paper describes designing ideas, the implementation of architecture and function in the Dawning 4000 Policy Based Autonomic Management System; then, it gives an objective evaluation of the Dawning 4000 Policy Based Autonomic Management System; at last, the paper summarizes the author's research and proposes future work.

Key Words：Cluster，Policy Based Management，Software Aging，Software Rejuvenation
