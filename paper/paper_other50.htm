高性能计算的性能评测与性能优化

摘　　要

高性能计算是一个国家综合实力的体现，被普遍认为是人类认识自然的三大支柱之一。目前，高性能计算的性能评价在高性能计算领域中显得越来越重要，性能评价和性能分析可以揭示高性能计算机、并行算法和并行应用程序的性能特点和性能瓶颈，指导高性能计算机、并行算法和应用程序的设计与改进。与此同时，机群系统以其良好的可扩展性和性能价格比，已迅速成为高性能计算领域的主流体系结构。

本文着眼于高性能计算机的体系结构，针对高性能应用程序的具体特点，从处理器、存储层次、通信系统、并行编程模式四个不同的层面出发，结合机群系统特别是曙光系列高性能计算机，采用理论分析和实际测试两方面相结合的方法，对高性能计算机、并行算法和并行应用程序作性能评价、性能分析和性能优化，指出各个层面的性能影响因子并研究其对性能影响的机理，提出了一些性能优化的方法。在处理器层面，分析了现代处理器中常用的一些技术，包括超线程技术、超长指令技术和NUMA架构等，对不同特点的程序的性能影响和如何充分利用处理器中的一些技术提升性能；在存储层次上，结合机群系统上Cache、主存、本地辅存及网络存储的存储层次的特点，分析了数据局部性、Cache命中率等对性能的影响以及提出如何提高数据局部性；通讯系统层面分析了不同机群通讯系统及不同机群通讯网络的特点；在编程模式方面，分析和比较了目前最常用两种并行编程模式--消息传递编程模式和共享存储编程模式，在机群系统上对性能的影响因子以及提出如何提高性能的方法。

本文的主要工作是从高性能计算机的体系结构出发，特别是从曙光系列高性能计算机的体系结构出发，结合高性能计算的特点，对高性能计算机、并行算法和并行应用程序进行性能评价和性能优化。

关键词：高性能计算，性能评测，机群，体系结构

Performance Evaluation and Optimization of
High Performance Computing

Abstract

The research capability of High Performance Computing (HPC) reflects the national composite power. The performance evaluation, analysis, and optimization of HPC is increasingly important in the HPC domain. Performance evaluation and analysis are used to discover the performance characteristics of high performance computers, parallel algorithms and parallel application programs, and locate performance bottlenecks. Performance evaluation and analysis plays an important role in directing the design and improvement of such systems. Clusters have become the mainstream architecture in parallel computing. This is due to the high degree of scalability and superior performance price comparison of these systems.

Based on the High Performance Computer Architecture and the characteristics of high performance applications, we investigated how the four key attributes - CPU, memory hierarchy, communication and parallel programming mode - affect the performance of HPC by means of both theoretical analysis and testing. In terms of the CPU attribute, some modern techniques, such as Hyper-threading, VLIW, NUMA architecture etc., and their respective performance characteristics in improving the CPU speed, have been studied. In terms of memory hierarchy, we investigated how to improve the cache hit rate and localize data according to the characteristics of cache, master memory, local disk and remote disk in cluster systems. As for communication, features of cluster systems and different cluster internal networks are examined. Two popular parallel programming models - message passing model and share memory model - are analyzed in terms of how do they affect the cluster performance and how can we improve the cluster performance.

Current research usually bases the performance analysis of HPC on algorithms. This work bases its performance analysis on the High Performance Computer Architecture, in particular the Dawning architecture.

Key Words：High Performance Computing，Performance Evaluation，Cluster，Computer Architecture
