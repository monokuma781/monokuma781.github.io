 机群通信子系统的研究，设计和实现

 
机群通信子系统的研究，设计和实现


机群就是由一组计算机系统（结点）通过高性能网络或局部网互联而形成的具有单一系统映象的高可用、高性能、高可扩展性的计算机集群系统。机群的发展与机群通信系统的发展是分不开的。机群的通信系统包括互联网络硬件以及通信软件。本文结合了作者在曙光2000机群系统通信软件的开发实践，对机群通信子系统进行了研究。本文首先对机群，机群的互联网络技术以及机群通信系统进行了广泛和深入的研究。然后分析了曙光2000的通信软件系统中不同数据通路的细节和特点，在此基础上设计和实现了曙光2000通信软件系统中的两大重要通信子系统。本文详细地介绍了这两个通信子系统设计的原理、技术以及实现的策略和方法，并对它们的性能进行了分析。

本文介绍的重点之一是基于快速以太网的底层通信软件的设计和实现。这个通信子系统是在曙光2000中，基于Digital的PCI快速以太网控制芯片21140A所设计和实现的底层通信软件。这个通信子系统的设计结合了曙光2000的基本通信库BCL，在快速以太网卡硬件上为BCL中提供一个高性能的虚拟通信处理机。该通信系统实现了面向BCL的用户级通信机制，并借助RISC的思想，为BCL提供了灵活的通信服务接口。

另外一个通信子系统是基于二维Mesh网的伪以太网卡设备驱动程序。为了让用户可以使用常用的通信编程接口（如socket编程接口）在二维Mesh网上进行通信，我们在PMI底层驱动程序的基础上扩展了一层软件接口。这层软件接口模拟了以太网卡驱动程序的功能，为上层的TCP/IP通信协议软件提供了数据链路的通信服务。

关键字：机群，机群通信，快速以太网，用户级通信，二维Mesh网，TCP/IP，通信接口。

A cluster is a group of high-performance computers connected by a high-performance interconnection network or a high-speed LAN to achieve high availability, high performance and high scalability with a single system image. The development of clusters is closely related to their communication systems which include the interconnection hardware and the communication software. This paper is based on the development practices on the Dawning 2000 cluster communication software.

In this paper, we first give an overview of clusters in the following aspects: the development trend, the architecture, and the main features. Then we discuss the interconnection techniques in a cluster , explore the details of the different communication paths and analyse the nature of these different paths in the communication systems of Dawning 2000 cluster. Based on this careful study we present two important communication subsystems designed by the author for the Dawning 2000 cluster. This paper gives the detail information on the design principles, techniques and implementation strategies and methods.

One of these two subsystems is the low-level communication layer on fast Ethernet. Using DEC 21140A , a fast Ethernet controller , as the underlying hardware, the subsystem provides a virtual communication machine for the basic communication library (BCL) which is a communication facility implemented for the Dawning 2000 cluster. This layer employs the memory mapping techniques to provide a user-level communication mechanism for the BCL. And the communication services provided by this low level communication layer are through a flexible interface which uses the idea of RISC for reference. In this section of the paper, we discuss the design scheme of the communication subsystem and come out with a virtual communication machine with a "RISC"-like interface. Then we explain the detail information about the structure of the subsystem and some important issues to be solved in the designing process. Finally, we list the communication interface and explain how to use it. And the detail information of DEC 21140A is discussed in the appendix.

The other subsystem is the pseudo Ethernet adapter device driver based on the 2-D mesh network. The purpose of this subsystem is to support the common communication interfaces (like socket interface) in the 2-D mesh interconnection network. The pseudo Ethernet adapter device driver is based on the PMI (PCI to Mesh Interface) device driver and provides Ethernet adapter simulation to the upper communication layer -- IP layer. In this section, we first explain the AIX's network architecture, which provides a guide to implement the network device driver. Then the two interfaces are discussed: the interface between the operation system and the pseudo Ethernet and the one between the pseudo device driver and the PMI device driver. Finally in this section we describe the architecture of the subsystem and provide some implementation strategies during the development process.

In the last chapter of this paper we make the performance analysis on the two subsystems discussed above and explain the results.

Keywords: clusters, cluster communication, fast Ethernet, user-level communication, 2-D mesh, TCP/IP, communication interface
