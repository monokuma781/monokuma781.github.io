 可扩展的单一系统映象机群文件系统的设计和实现

 
可扩展的单一系统映象机群文件系统的设计和实现


高性能计算技术和高性能服务器技术的一个发展趋势是利用商品化部件构成机群体系结构。该结构强调系统的可扩展性、单一系统映象、好用易管理性，以及高可用性。曙光2000是一个典型的以机群体系结构设计的超级服务器系统。我们为曙光2000设计和实现了可扩展的单一映象文件系统──COSMOS。本文描述了COSMOS的设计、实现和性能分析，着重讨论了其中的一些关键性问题。

COSMOS首先是一个单一映象的分布式文件系统，支持数据的透明存取和共享访问。机群中各个结点上看到的COSMOS目录结构和文件数据全局一致，并符合UNIX的文件共享语义。另外，我们在不修改AIX操作系统源码的前提下，通过Vnode/VFS层核心扩充，实现了与其底层平台的无缝连接，保证了与UNIX应用程序的完全二进制兼容。

COSMOS的客户端实现了合作式缓存以提高系统的性能。我们设计了基于令牌的写使失效协议来维护缓存一致性，并提出了双粒度的一致性优化协议来减少开销。除了数据外，我们分别为inode、目录和超级块设计了优化的一致性协议，并证明了协议满足缓存一致性。

COSMOS还是一个可扩展的分布式系统，在设计上采用无集中服务器的Serverless结构，将数据存储和元数据管理工作分布到整个系统中，并实现了分组的stripe存储，提高了系统的带宽和扩展性。

测试表明，COSMOS系统带宽和综合性能良好，与NFS相比有明显优势，且具有良好的可扩展性，基本上达到了设计目的。目前COSMOS即将应用于曙光2000-II机群系统。除了产品化方向的努力外，COSMOS也为我们对分布式文件系统的进一步研究提供了一个良好的平台。

关键词：机群，分布式文件系统, 单一系统映象，可扩展性，无集中式服务器，存储分组，合作式缓存

A current trend in high performance computing and superservers is to build clusters with commodity components. Cluster architecture emphasis on its high scalability, single-system image, usability, manageability and high availability. Dawning 2000 is a typical superserver built by clustering commodity workstations. We designed and implemented its scalable single-image file system, COSMOS. This thesis describes the design, implement-ation and performance analysis of COSMOS.

First, COSMOS is a distributed file system with support for single-system image. File data can be accessed and shared transparently through network. All nodes in the cluster see the same global directory hierarchy of COSMOS. UNIX semantics is enforced for file data and metadata. We implemented seamless extension of OS for COSMOS with Vnode/VFS kernel interface.

Cooperative cache is implemented on COSMOS clients to improve performance. We designed a token-based write-invalidate protocol to maintain cache coherence. And we proposed an optimized protocol of bi-granularity consistency to lower protocol overhead.

Second, COSMOS is designed to be a scalable system. COSMOS utilizes the serverless architecture: all nodes in cluster cooperate to provide file system service. No bottle-neck of centralized server limits the scalability. Data storage and meta-data management is distributed in the whole system for high performance and scalability. Striped storage is used to provide collective bandwidth.

Our test shows satisfactory bandwidth and overall performance of COSMOS, much better than that of NFS; the system scales well. COSMOS have met the design goals, and will be running on Dawning 2000-II cluster in near future. Besides efforts towards commercial product, COSMOS provides a good platform for our further research on distributed file systems.

Key words: distributed file system, single-system image, scalability, serverless, striped storage, cooperative cache
