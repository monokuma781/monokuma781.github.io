 可扩展I/O的研究和参考实现

 
可扩展I/O的研究和参考实现


在文件系统和I/O领域，提高性能和I/O的可扩展性是人们的研究目标。本文着重于一种可扩展I/O标准界面的设计、一个可扩展文件系统的设计，以及在大规模并行处理系统和机群体系结构系统上参考实现和原型系统的实现与评价。本文提出一种新型的可扩展I/O底层API（SIO-LLAPI），这种简单的以性能为目标的底面界面是用以支持文件系统开发者有效地开发高层的并行I/O界面和为并行I/O的研究者提供基础平台。本文描述了在Intel Paragon多机系统上SIO-LLAPI的一种参考实现，其中，我们提供了与Intel并行文件系统（PFS）兼容的文件系统结构和文件条块化算法，支持SIO-LLAPI的所有核心特色和部分可选特色。性能评价表明，SIO-LLAPI提供了大幅度提高I/O性能的可能性，并且易于实现，它的API灵活性对并行I/O的研究很有帮助。我们在SIO-LLAPI之上实现了Intel PFS面，实验结果表明，在大多数情况下，性能与原始的PFS性能相当，在某些情况下，比原始的PFS性能更稳定和更好。本文还描述了COSMOS可扩展网络文件系统的设计，它采用虚拟文件系统扩展技术提供了单一文件系统映像，实现与UNIX文件系统的无缝连接，UNIX的文件操作语义通过文件系统的保持全局一致的客户端缓存保证，采用Server-less体系结构，支持软件RAID用以提高I/O带宽，支持分布式管理策略用以平衡系统的负载。我们在曙光2000超级计算机上实现了一个SIO界面的原型系统，它完全位于用户空间库中，与COSMOS紧密结合。我们设计了一个全局的Directory Daemon用以维护文件系统的一致性。为了模块化的考虑，我们将原型系统分成两个层次，一个层次与API相关，另一个层次用来处理文件系统结构和实际数据存取。原型系统性能能够与COSMOS相当，在路径名等操作上甚至更好。

In the field of file system and I/O system, it is the research object to improve the performance and scalability. This thesis puts emphasis on the design of a standard scalable I/O interface, the design of a scalable file system, the implementation and evaluation of a reference implementation on the massively parallel processing system and the prototype on the system with cluster architecture. This dissertation presents a new Scalable I/O Low-level API (SIO-LLAPI), that provides file system implementers with a simple and performance- driven low-level interface to support high-level parallel I/O interfaces efficiently and effectively, and provides parallel I/O researchers the fundamental platform. In this dissertation we describes a reference implementation of SIO-LLAPI on the Intel Paragon multicomputer. The implementation provides the file system structure and file striping algorithm compatible with Intel Parallel File System (PFS). The implementation supports the core features and some optional features of the low-level API. Our preliminary performance evaluation with the implementation shows that the SIO-LLAPI provides opportunities of significant performance improvement, is easy to implement, and helps the research of parallel I/O with API flexibility. We have implemented an Intel PFS interface on top of the SIO low-level API. Our experiments show that, in most cases, the performance of such an implementation is as good as that of Intel's native PFS. In other cases, the SIO implementation is much more stable and better than that of the PFS. The dissertation also illustrates the design of COSMOS scalable network file system. It provides single file system image by virtual file system extension technique, and the seamless connection with Unix file system. The Unix file semantics is maintained by client cache coherence of file system and Unix-like I/O atomicity. It has the server-less architecture. Further, it supports the software RAID to improve I/O bandwidth, and uses management distribution policy to balance the load. We implemented a SIO prototype cooperating closely with COSMOS file system on Dawning-2000 supercomputer. The prototype is totally on the user level library. But we designed a global Directory Daemon to keep file system structure consistent with COSMOS. We divided the prototype into two layers for modularity purpose, one is relevant to the API, and another is to handle the file system structure and data access. On Dawning-2000, the performance is as good as that of COSMOS, and even better for the pathname operations
