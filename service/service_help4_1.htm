ESSL的使用

AIX工程与科学子程序库ESSL3使用指南

　　针对不同的RS/6000序列处理器，ESSL 3提供了四个版本的子程序库：

  • ESSL SMP Library. 此版本适合RS/6000 SMP处理器（如604和604e），支持多线程。曙光2000-II采用的正是此类处理器，每个节点有2个共享存储的CPU。
  • ESSL Thread-Safe Library. 此版本适合所有RS/6000 处理器，用户可利用此库开发自己的多线程应用程序。
  • ESSL POWER Library. 此版本适合RS/6000 POWER 和PowerPC处理器。
  • ESSL POWER2 Library. 此版本适合RS/6000 POWER2 处理器。

　　上述四个版本的子程序库的计算结果在数学上是等价的。面向曙光2000-II的应用程序可调用ESSL的前3个版本。若使用了多线程，则必须调用ESSL的前两个版本。

　　值得注意的是，调用不同版本的ESSL的应用程序，进行编译时要用相应的编译器和编译选项。具体情况可参见本指南的“程序编译”部分。

　　ESSL3提供了458个子程序，涉及以下10个领域：

1.线性代数子程序

  • 向量--标量运算
  • 稀疏向量--标量运算
  • 矩阵--向量运算
  • 稀疏矩阵--向量运算

2.矩阵操作

  • 矩阵加
  • 矩阵减
  • 矩阵乘
  • 秩-k修正
  • 秩-2k修正
  • 矩阵转置

3.线性代数方程组

  • 稠密线性代数方程组
  • 带状线性代数方程组
  • 稀疏线性代数方程组
  • 线性最小二乘

4.特征值问题

  • 标准特征值问题
  • 广义特征值问题

5.信号处理

  • 傅立叶变换
  • 卷积与相关性
  • 相关的计算

6.排序和查找

  • 排序和带索引排序
  • 二分和顺序查找

7.插值

  • 多项式插值
  • 立方样条插值

8.数值积分：关于点集或函数的数值积分

9.随机数发生器

  • 产生均匀分布的随机数
  • 产生正态分布的随机数

10.工具：一般的服务操作

 

Fortran程序调用ESSL

调用格式

　　ESSL子程序遵循标准Fortran的调用习惯。因此，Fortran程序调用ESSL库是非常方便的。ESSL子程序可分成两类：过程和函数。过程类子程序用CALL语句调用：

　　CALL subroutine-name (argument-1, . . . , argument-n)

　　例如，调用子程序SAXPY，可用如下语句：

　　CALL SAXPY (5, A, X, J+INC, Y, 1)

　　函数类子程序的调用，与通常的Fortran 函数一样：可以用它给变量赋值，也可以把它作为表达式的一部分。但要注意：在Fortran程序中要声明ESSL函数返回值的类型。例如，要调用ESSL函数DASUM，程序中应有如下相应语句：

　　DOUBLE PRECISION DASUM,SUM,X
　　.
　　.
　　.

　　SUM = DASUM (N,X,INCX)

多线程编程

　　曙光2000-II采用RS6000/SMP处理器。使用多线程技术，可以充分利用SMP处理器的结构特点，获取高性能。下面给出一个使用多线程的Fortran 程序实例。该程序最多可以产生8个线程，每个线程调用ESSL的DURAND和DGEICD子程序。

　　program matinv_example
　　implicit none
　　!
　　! program to invert m nxn random matrices
　　!

　　real(8), allocatable :: A(:,:,:), det(:,:), rcond(:)
　　real(8) :: dummy_aux, seed=1998, sd
　　integer :: rc, i, m=8, n=500, iopt=3, naux=0
　　!
　　! allocate storage
　　!

　　allocate(A(n,n,m),stat=rc)
　　call error_exit(rc,"Allocation of matrix A")
　　allocate(det(2,m),stat=rc)
　　call error_exit(rc,"Allocation of det")
　　allocate(rcond(m),stat=rc)
　　call error_exit(rc,"Allocation of rcond")

　　!
　　! Calculate inverses in parallel
　　!
　　!SMP$ parallel do private(i,sd), schedule(static),
　　!SMP$& share(n,a,iopt,rcond,det,dummy_aux,naux)
　　do i=1,m

　　sd = seed + 100*i
　　call durand(sd,n*n,A(1,1,i))
　　call dgeicd(A(1,1,i),n,n,iopt,rcond(i),det(1,i),
　　& dummy_aux,naux)
　　enddo

　　write(*,*)'Reciprocal condition numbers of the matrices are:'
　　write(*,'(4E12.4)') rcond
　　!
　　!
　　!
　　deallocate(A,stat=rc)

　　call error_exit(rc,"Deallocation of matrix A")
　　deallocate(det,stat=rc)
　　call error_exit(rc,"Deallocation of det")
　　deallocate(rcond,stat=rc)
　　call error_exit(rc,"Deallocation of rcond")
　　stop

　　contains
　　subroutine error_exit(error_code,string)
　　character(*) :: string
　　integer :: error_code
　　if(error_code .eq. 0 ) return
　　write(0,*)string,": failing return code was ",error_code
　　stop 1
　　end subroutine error_exit
　　end

　　如前所述，上述程序必须用带 -qnosave 选项的编译命令xlf_r进行编译，另外由于采用了!SMP$类制导性语句，此程序的编译还必须使用-qsmp选项。
 

C程序调用ESSL库

调用格式

　　对于过程类ESSL子程序的调用，C程序采用如下语句：

　　　　subroutine-name (argument-1, . . . , argument-n)

　　例如，可用下面的语句调用子程序 SAXPY :

　　　　saxpy (5,a,x,incx,y,1)

　　对于函数类ESSL子程序的调用，C程序采用如下语句：

　　　　return-value =subroutine-name (argument-1, . . . , argument-n)

　　例如，可用下面的语句调用子程序DASUM ：

　　　　sum = dasum (n,x,incx);

与Fortran调用ESSL库的主要区别

　　C程序调用ESSL与Fortran程序有两点主要区别：

　　(1) C程序必须包含ESSL头文件

　　　　C程序调用ESSL库时，必须包含ESSL头文件。即在源程序应有如下语句：

　　　　　　# include<essl.h>

　　　　如果使用ESSL Thread-Safe或 SMP版本，还必须有如下语句：

　　　　　　# include<pthread.h>

　　(2)数组存储模式不同

　　　　C语言中的数组按行存储，而ESSL子程序要求数组按列传递参数。用户可以采取以下任意一种措施，解决这一问题：

  • 从一开始就用(逻辑上的)数组的转置构造数组，若有必要，将结果转置。
  • 调用ESSL之前，将矩阵转置；调用ESSL之后，再将结果矩阵转置。
  • 使用ESSL子程序的矩阵转置标志"t"。例如，假设a，b都是n x n 矩阵，为了计算c = a b, 可以用下列方式调用ESSL子程序sgemul:
    sgemul ( a, n, "t", b, n, "t", c, n, n, n, n)

C程序中使用复数和逻辑型数据

　　由于C语言本身不含复数和逻辑数据类型，ESSL在其头文件essl.h中为用户定义了两个数据类型：cmplx 和dcmplx. 它们分别对应短精度复数类型和长精度复数类型。另外，ESSL 还定义了两个宏：RE 和RM ，分别处理复数的虚部和实部。定义代码段如下：

　　#ifndef _CMPLX
　　#ifndef _REIM
　　#define _REIM 1
　　#endif
　　typedef union { struct { float __re, __im;}
　　__data; double __align;} cmplx;
　　#endif
　　#ifndef _DCMPLX
　　#ifndef _REIM
　　#define _REIM 1
　　#endif
　　typedef union { struct { double __re, __im;}
　　__data; double __align;} dcmplx;
　　#endif
　　#ifdef _REIM
　　#define RE(x) ((x).__data.__re)
　　#define IM(x) ((x).__data.__im)
　　#endif

　　为了使用方便，用户可以定义一个CSET宏，利用RE和RM 这两个宏来初始化复数（cmplex 或 dcmplex）类型的变量。举例如下：

　　#include <essl.h>
　　#define CSET(x,a,b) (RE(x)=a, IM(x)=b)
　　main()
　　{
　　cmplx alpha,t[3],s[5];
　　.
　　.
　　.
　　CSET (alpha,2.0,3.0);
　　caxpy (3,alpha,s,1,t,2);
　　.
　　.
　　.
　　}

　　如果用户选择使用自己定义的复数类型，则必须在程序中定义_CMPLX 和_DCMPLX.在#include<essl.h>语句之前，必需有如下定义：

　　#define _CMPLX
　　#define _DCMPLX

　　至于逻辑型数据，用户可作如下定义：

　　#define FALSE 0
　　#define TRUE 1

　　这样TRUE 和FALSE 就可以作为逻辑型数据传递给ESSL子程序的逻辑型参数。

多线程编程

　　曙光2000-II采用RS6000/SMP处理器。使用多线程技术，可以充分利用SMP处理器的结构特点，获取高性能。下面给出一个使用多线程的C程序实例。该程序产生两个线程，每个线程调用ESSL的子程序ISAMAX：

　　#include <pthread.h>
　　#include <stdio.h>
　　#include <unistd.h>
　　#include <essl.h>

　　/* Create structure for argument list */
　　typedef struct {
　　int n;
　　float *x;
　　int incx;
　　} arg_list;

　　/* Define prototype for thread routine */
　　void *Thread(void *v);

　　int main()
　　{
　　float sx1[9] = { 1., 2., 7., -8., -5., -10., -9., 10., 6. };
　　float sx2[8] = { 1.,12., 7., -8., -5., -10., -9., 19.};
　　pthread_t first_th;
　　pthread_t second_th;
　　int rc;
　　arg_list a_l,b_l;

　　/* Creating argument list for the first thread */
　　a_l.n = 9;
　　a_l.incx = 1;
　　a_l.x = sx1;

　　/* Creating argument list for the second thread */
　　b_l.n = 8;
　　b_l.incx = 1;
　　b_l.x = sx2;

　　/* Creating first thread which calls the ESSL subroutine ISAMAX */
　　rc = pthread_create(&first_th, NULL, Thread, (void *) &a_l);
　　if (rc) exit(-1);

　　/* Creating second thread which calls the ESSL subroutine ISAMAX */
　　rc = pthread_create(&second_th, NULL, Thread, (void *) &b_l);
　　if (rc) exit(-1);

　　sleep(1);
　　exit(0);
　　}

　　/* Thread routine which call ESSL routine ISAMAX */
　　void *Thread(void *v)
　　{
　　arg_list *al;
　　float *x;
　　int n,incx;
　　int i;

　　al = (arg_list *)(v);
　　x = al->x;
　　n = al->n;
　　incx = al->incx;


　　/* Calling the ESSL subroutine ISAMAX */

　　i = isamax(n,x,incx);
　　if ( i == 8)
　　printf("max for sx2 should be 8 = %d\n",i);
　　else
　　printf("max for sx1 should be 6 = %d\n",i);
　　}

ESSL库函数性能

ESSL库函数的编程技巧

　　性能是设计ESSL库函数的主要目的。为达到该目的，ESSL库函数采用适于硬件设计特性的先进算法，这些特性包括高速缓存的尺寸，TLB尺寸，页面尺寸，可用寄存器的数目等等。ESSL库函数大部分用FORTRAN来执行，并且尽可能采用乘加指令。

　　多数ESSL库函数采用下述技术来优化性能：

  • 有效管理高速缓存和TLB以使其命中率最大化,也就使尽量是计算所需的数据驻留在高速缓存和TLB中。
  • 被访问的数据应在连续的存储空间中，即应使用步长为1的数组计算。
  • 充分利用浮点寄存器。
  • 采用换页极小的算法。
  • ESSL SMP库的设计目的是充分利用处理器的处理能力和共享内存。此外，ESSL SMP一部分子函数的编码充分利用了多线程和并行编程技术以获得高性能。其他一些ESSL SMP子函数一次或多次调用这些多线程子程序，也会从高性能中获益。
  • 对线程数目的选取取决于问题的大小，特定子程序的调用，及运行的物理处理器个数。 要达到最佳性能，试验是必需的。
  • 所有的ESSL库函数尽量采用先进的数学方法以达到高性能，如矩阵--向量线性代数子程序操作基于高层数据结构，基于矩阵--向量，而非向量--标量。其结果是它们直接或间接优化了调用ESSL函数的程序。 　

ESSL库函数的合理使用

　　在调用ESSL库函数时应注意以下几点问题, 以充分发挥其高性能的特点：

  • 作基本线性代数和矩阵运算时，应优先调用ESSL 库的矩阵--矩阵计算类子函数，其次是矩阵--向量计算类子函数，再次是向量--标量子函数。
  • 必要时调用作多次运算的子函数，如 SNDOT和SNAXPY , 而不是多次调用只作单个运算的子函数，如SDOT和SAXPY。
  • 尽量使用步长为1的数组计算。
  • 一个数组的头维数（LDA）或一个向量相邻元的间距（INC）不要定义为等于或接近128（双精度数组）或256（单精度数组）的倍数。
  • 避免将一维数组的长度定义为128的倍数。
  • 对小规模的问题，避免将数组的头维数定义得很大。

　　不论是哪种数据结构，尽量将数组长度定义为双字长的倍数。
